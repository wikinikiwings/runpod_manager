# UI / design conventions (inline `FRONTEND_HTML`)

Весь фронтенд — одна строка `FRONTEND_HTML` в `runpod_manager.py` (HTML + CSS +
JS). Темизация тёмная, через CSS-переменные (`--bg`, `--c1`, `--t`, `--t2`,
`--t3`, `--ac`, `--bd`, `--rs`, `--mono`). Этот документ фиксирует соглашения,
чтобы новые элементы не выпадали из общего дизайна.

## Поля ввода — ВСЕГДА указывать `type`

Главное правило (и самая частая ошибка). Тёмная тема навешивается на инпуты
**по атрибуту `type`**, а не по тегу/классу:

```css
input[type=text],input[type=password],input[type=number],input[type=time],input[type=date],select{
  font-family:var(--mono);font-size:13px;background:var(--bg);color:var(--t);
  border:1px solid var(--bd);border-radius:var(--rs);padding:8px 12px;width:100%;outline:none}
input:focus,select:focus{border-color:var(--ac)}
```

Селектор `input[type=text]` **не матчит** `<input>` без атрибута `type` —
такой инпут получает дефолтный браузерный белый стиль и визуально выпадает из
тёмной темы.

- ✅ `<input type="text" class="...">`
- ❌ `<input class="...">` (станет белым — был баг блока «🖼 Образы подов»,
  исправлен 2026-06-25)

`<select>` темизируется автоматически (есть в селекторе) — отдельный `type`
ему не нужен.

При добавлении нового поля: задавай явный `type` (`text`/`password`/`number`/
`time`/`date`) — стиль подхватится сам, своего CSS писать не нужно.

## Структура полей

- Оборачивай пару «подпись + контрол» в `<div class="fr">…</div>` (даёт
  вертикальный ритм `margin-bottom:14px`).
- Подпись — `<label>…</label>` (стилизуется автоматически: моно, 11px,
  `--t3`, блочная).
- Тумблеры — `<label class="toggle"><input type="checkbox" …><span class="sw">
  </span> Текст</label>` (кастомный свитч; нативный чекбокс скрыт).
- Сетки одинаковых полей (квоты, оффсеты, образы по проектам) — контейнер
  `class="quota-grid"`.

## Текст и акценты

- Приглушённый текст/подсказки — `var(--t3)` (или `--t2`), размер 11–13px,
  `font-family:var(--mono)`.
- Акцентный цвет (фокус, активный тумблер) — `var(--ac)`.
- Пользовательский ввод, попадающий в HTML, **экранировать** (см. `imgEsc()` в
  блоке образов): минимум `&`, `<`, `>`, `"`.

## Кнопки

- Базовый класс `.btn`; модификаторы вида `.bs`/`.bp` для размеров/основной
  кнопки (см. существующие вызовы).
